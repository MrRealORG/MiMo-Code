Thanks for looking at these. I think the #526 part is directionally right, but this PR may be easier to review if it is split by issue. A few concrete notes:

I use screen , then in screen when to open mimo, I cannot type ! #526: the screen/tmux detection should probably have a focused regression test for TERM=screen*, STY, TERM=tmux*, TMUX, and the explicit MIMOCODE_TUI_PLAIN=false opt-out.
响应的完整内容会输出到状态栏中 #502: status truncation is also covered in fix: actor tool stringified ops, model search, custom provider UX, editor cursor bug #570, so one of these PRs will likely become duplicate.
无法读取word文档 #513: the .docx path is higher-risk than the TUI fixes. execSync(unzip -p "${filepath}" ...) goes through a shell and can break or become unsafe for paths containing quotes. It also only handles .docx, while the issue text mentions PDF too. If this stays, it probably needs either spawn/execFile-style argument passing or a zip/XML parser approach plus a focused fixture test.
My suggestion: keep the terminal keyboard fix as a small #526-only PR first, with tests, and move .docx reading into a separate PR once the exact file support scope is clear.
